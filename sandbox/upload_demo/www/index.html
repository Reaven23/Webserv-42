Webserv Upload Demo

Upload, browse & delete files — GET / POST / DELETE

Upload Browse Delete Errors

POST Upload a file

Choose a file from your computer


The file is sent as multipart/form-data. The server parses the boundary, extracts the filename and binary data, and writes it to the upload_path directory. After upload you will see "201 Created" — click Back to return here, then check Browse to see your file.

GET Browse uploaded files

Click to open the uploads directory (autoindex) :

Or preview here (iframe) :

Click any filename in the list to view/download it. Images will display in the browser, other files will download.

DELETE Delete files

HTML forms don't support the DELETE method. Use curl from your terminal :

# Delete a specific file
curl -i -X DELETE http://localhost:8080/uploads/photo.jpg

# Delete again (already gone) -> 404
curl -i -X DELETE http://localhost:8080/uploads/photo.jpg

Replace photo.jpg with the actual filename. Refresh the Browse section to see it disappear.

ERRORS Test error responses

From the browser :

  • 404 GET /does_not_exist
  • 404 GET /uploads/ghost.txt
  • 403 Path traversal

From curl :

# 405 - POST on / (only GET allowed)
curl -i -X POST -d "test" \
  http://localhost:8080/

# 413 - Body too large (> 10KB)
curl -i -X POST \
  -d "$(python3 -c "print('A'*20000)")" \
  http://localhost:8080/uploads/big.txt

# 400 - Null byte in URI
curl -i http://localhost:8080/%00test

# 403 - DELETE a directory
curl -i -X DELETE \
  http://localhost:8080/uploads/
